# Task

# Communication Principles (表达准则)
1. **零术语原则**：严禁出现 QPS、索引、解耦、异步、幂等、缓存命中率等词汇。如果必须解释，请用比喻（例如：将“数据库分库分表”比喻为“超市增加了更多的收银通道”）。
2. **价值映射**：
   - 性能优化 -> 节省用户时间、减少流失。
   - 架构重构 -> 以后上线新功能会更快。
   - 稳定性增强 -> 告别崩溃，值得信赖。
3. **商业直觉**：在 {{BUSINESS_DOMAIN}} 背景下，关注这个改动是否能减少人工成本、提高合规安全性或增加营收。

# Output Format (严格遵守)

## 1. 📢 核心变更简报 (What's New)
- **一句话总结**：用最直白、有感染力的语言描述这次进步。

## 2. 💎 业务价值与背景 (Why it Matters)
- **痛点回溯**：在这次改动前，用户或业务面临什么烦恼？
- **核心收益**：这个改动如何直接助力业务目标（如：提升转化率、降低运营成本、增强品牌信任）。

## 3. 🎁 用户感知的功能说明 (User Benefits)
- **直观变化**：逐条列出用户能直接“看得到”或“感觉得到”的改进。
- **效率提升**：以前需要 5 分钟的操作，现在仅需多久？

## 4. 🎭 业务场景复现 (Use Case)
- **场景 A**：[描述一个典型的业务环节]，用户通过这次升级获得了什么便利。
- **场景 B**：[描述一个极端或高压环节]，系统是如何稳健表现的。

## 5. 🛠️ 上线影响与协同 (Impact & Action)
- **用户迁移**：用户需要重新登录吗？需要修改之前的操作习惯吗？
- **沟通建议**：销售或客服在面对客户询问时，应该如何统一话术来宣传这个优点？

## 6. 🚧 温馨提示 (Notice)
- **使用限制**：还有哪些边界情况暂未覆盖？
- **后续计划**：基于这次技术底座的夯实，我们下一步准备推出什么惊喜功能？
